# v0.2.3 Release Notes

## English

### Why this release

- `v0.2.3` closes the remaining documentation and version drift around the current OneSync mainline.
- The release baseline now matches the shipped settings-center redesign, the primary `Improve All Skills` workflow, and the scope-aware AstrBot local skill surface.

### Core capabilities highlighted

- Embedded WebUI at `127.0.0.1:8099` with a clearer settings center built around summary cards and grouped operator controls.
- Aggregate-first Skills management with one primary `Improve All Skills` action, shared progress reporting, and replayable execution history.
- Scope-aware AstrBot local skill operations across `global / workspace`, backed by explicit runtime layout and summary fields.

### User convenience highlights

- Operators can read current mode, polling interval, WebUI port, and password state without parsing a long form first.
- The main Skills action rail is simpler: improve-able atoms and actionable aggregates are now described as one guided workflow instead of separate mental models.
- API and maintainer docs now line up with the real AstrBot runtime contract, so frontend/API consumers do not have to infer scope behavior.

### Upgrade notes

- No breaking config migration in this release.
- If your browser still shows an older WebUI bundle, do a hard refresh after upgrading.
- If you run the embedded WebUI through AstrBot, restart AstrBot once after pulling the new version.

### Verification

- `python3 -m py_compile main.py skills_core.py skills_hosts_core.py skills_astrbot_state_core.py skills_astrbot_actions_core.py skills_runtime_health.py webui_server.py` -> passed
- `pytest -q` -> `204 passed in 1.00s`
- `git diff --check` -> clean

## 中文

### 这个版本解决什么问题

- `v0.2.3` 主要收口当前 OneSync 主线的文档与版本漂移。
- 这次版本把已经落地的设置中心重构、`一键完善 Skills` 主流程，以及 AstrBot 本地 skill 的 scope-aware 管理正式固化到发布基线里。

### 重点能力

- 内置 WebUI 继续运行在 `127.0.0.1:8099`，设置中心改成摘要卡 + 分组控制，更适合运维阅读。
- Skills 管理坚持聚合优先，并以 `一键完善 Skills` 作为主操作，统一承接进度、报告和历史。
- AstrBot 本地 skill 动作现已覆盖 `global / workspace` 两类范围，并由明确的运行态 layout / summary 字段支撑。

### 用户便利性改进

- 用户无需先翻长表单，就能直接看到当前模式、轮询周期、WebUI 端口和口令状态。
- Skills 主操作更简单：待改善 install atom 与可执行聚合被收成一条流程，不再需要用户自己拼接操作顺序。
- API 文档、维护文档与前端行为已对齐到同一份 AstrBot scope 合同，联调和排障成本更低。

### 升级说明

- 本版本没有破坏性配置迁移。
- 如果浏览器仍显示旧版 WebUI，请升级后执行一次强制刷新。
- 如果你通过 AstrBot 运行内置 WebUI，请在拉取新版本后重启一次 AstrBot。
