# v0.2.2 Release Notes

## English

### Why this release

- `v0.2.2` turns the recent OneSync polish into a clearer public baseline: easier onboarding, cleaner documentation boundaries, and a lower-noise Skills operations surface.

### Core capabilities highlighted

- Unified AstrBot software updater with scheduled checks, manual updates, verification, state persistence, and audit history.
- Independent WebUI at `127.0.0.1:8099`, so operators get a dedicated control plane without patching the AstrBot dashboard.
- Aggregate-first Skills management that treats install units, collection groups, source bundles, deploy targets, and host software as maintainable operational objects instead of flattening everything into leaf-level noise.

### User convenience highlights

- The top-level README now behaves like a real user homepage: core features, quick start, common commands, WebUI highlights, and skills-management guidance are surfaced first.
- Developer extension details and API details are now split into dedicated bilingual docs, so user docs stay concise while engineering docs stay deep.
- In the Skills inventory, `结构与成员` is collapsed by default to keep the main panel focused on primary management actions instead of secondary detail.
- Recent aggregate progress, execution replay, and freshness authority writeback are now part of the documented mainline story rather than hidden implementation detail.

### Upgrade notes

- No breaking change in this release.
- If your browser still shows an older WebUI bundle, do a hard refresh after upgrading.
- If you run the embedded WebUI through AstrBot, restart AstrBot once after pulling the new version.

### Verification

- `python3 -m py_compile main.py skills_core.py webui_server.py`
- `pytest -q` -> `191 passed`

## 中文

### 这个版本解决什么问题

- `v0.2.2` 将最近一轮 OneSync 打磨固化为更清晰的公开基线：接入说明更明确、文档分层更清楚、Skills 运维面板噪音更低。

### 重点能力

- 面向 AstrBot 的统一软件更新器，覆盖定时检查、手动更新、更新验证、状态持久化与审计回放。
- 独立 WebUI 运行在 `127.0.0.1:8099`，无需改 AstrBot Dashboard 即可得到专用运维控制面。
- Skills 管理坚持聚合优先，把 install unit、collection group、source bundle、deploy target 与宿主软件都视为可维护的运维对象，而不是把所有叶子 skill 平铺成噪音。

### 用户便利性改进

- 顶层 README 现在真正承担用户首页角色，优先展示核心特色、快速开始、常用命令、WebUI 亮点与 Skills 管理说明。
- 开发扩展说明与接口说明已拆分到独立的中英文文档，用户说明保持简洁，工程说明保持完整。
- Skills 清单中的 `结构与成员` 已默认收起，让主面板聚焦在高频管理操作，而不是次级结构细节。
- 最近补齐的聚合进度展示、执行结果回放与 freshness 权威回写，现在已被明确纳入主线能力说明，不再是隐藏实现细节。

### 升级说明

- 本版本没有破坏性变更。
- 如果浏览器仍显示旧版 WebUI，请升级后执行一次强制刷新。
- 如果你通过 AstrBot 运行内置 WebUI，请在拉取新版本后重启一次 AstrBot。

### 验证结果

- `python3 -m py_compile main.py skills_core.py webui_server.py`
- `pytest -q` -> `191 passed`
